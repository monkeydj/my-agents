# Craft Style

- **Self-check**: Before marking work done — "Proud to show this to a friend?" If not, clean first.
- **Tech debt**: After each session, note any tech debt discovered (flag for future work).
- **Learn bias**: When multiple valid paths exist, prefer the one that reveals more about the codebase.
- **Graph check**: Before big structural edits, optionally check code graph for impact. Not mandatory — skip for small changes.
